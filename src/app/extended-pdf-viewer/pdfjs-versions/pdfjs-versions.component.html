Most versions of ngx-extended-pdf-viewer ship with two versions of pdf.js:
  • The assets folder contains the latest release of pdf.js.
  • The bleeding-edge folder contains a developer build of pdf.js. That code is newer than the latest release, but it's not an officially supported version, so it's primarily there for testing purposes.
However, version 18 only offers the stable branch 3.10 of my branch of pdf.js. I've removed the bleeding edge branch because I've already started publishing preview versions of version 19, which features pdf.js 4.0.
